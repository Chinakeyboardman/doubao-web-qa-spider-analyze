### 评估结论

| 项目 | 推荐方式 | 理由 |
|------|----------|------|
| **Douyin_TikTok_Download_API**（抖音视频下载） | **独立部署** | 已有 Docker 镜像、FastAPI、完整 HTTP API；其他项目通过 `http://host:port/douyin/web/...` 调用即可；反爬算法（X-Bogus/A_Bogus）自研成本高 |
| **douyin-crawler**（豆包web爬虫） | **代码引用 / 子模块** | 非纯 API 服务，是 Node+Python+PostgreSQL+Redis 的完整 pipeline；需作为子项目纳入父项目，按业务改造 `worker/tasks.py` 等；无 pip 包，不适合独立部署后 HTTP 调用 |

### 不推荐自研全流程
- 两个项目均涉及平台反爬逻辑，自研周期长、维护成本高
- 现有开源方案可直接复用，优先「站在巨人肩膀上」

### 实施建议
1. **抖音视频下载**：独立部署 Douyin_TikTok_Download_API，父项目 `docker-compose` 中声明该服务，其他子项目通过环境变量配置其 API 地址
2. **豆包web爬虫**：以 git submodule 或直接目录形式纳入父项目，在现有 pipeline 上做业务适配（如对接豆包页面、对接 data-clean 的 DB）
