# 抖音视频内容获取：最低成本 + 高效方案

> 目标：拿到视频内说的内容（字幕或语音转文字），成本最低、效率最高

---

## 一、两种内容来源对比

| 方式 | 成本 | 效率 | 适用场景 |
|------|------|------|----------|
| **字幕** | 免费 | 高（API 直接返回） | 视频有 CC 字幕时 |
| **语音转文字 (STT)** | ~0.013 元/分钟 | 中（需下载+抽音频+ASR） | 无字幕时 |

---

## 二、推荐策略：字幕优先，STT 兜底

```
1. 先查 8080 API / hybrid 返回 → 若有 video_subtitle / caption_info，直接使用（免费）
2. 再查 douyin_videos.raw_data → 若有 subtitles，直接使用（免费）
3. 都无 → 走 STT 流程（下载 → 抽音频 → 火山 ASR）
```

### 当前数据现状（来自前期统计）

- 抖音链接 1,436 条，有 qa_link_content 的 881 条
- **无字幕**：100%（当前数据源均未提供字幕）
- **无内容且无字幕**：84.1%
- **有可用内容**：约 140 条（约 9.7%）

说明：已验证 8080 hybrid API 返回结构：
- `caption`：视频标题/描述（文案），可直接用
- `is_subtitled`：0=无字幕，1=有字幕
- `video_text`：有字幕时的字幕内容，当前样本为 `[]`
- 多数抖音短视频无内置字幕，`is_subtitled=0`、`video_text=[]` 时需走 STT

---

## 三、实现方式

### 方案 A：先验证 8080 是否返回字幕（推荐先做）

调用 `GET /api/hybrid/video_data?url=xxx`，检查返回的 `aweme_detail` 中是否有：

- `video_subtitle`
- `caption_info` / `captionInfos`
- `subtitles`

若有，在 `douyin_data_merger` 或 crawler 中提取并写入 `qa_link_content`，**零成本**。

### 方案 B：STT 兜底（已实现）

无字幕时，使用现有流程。**优先走视频**：模型支持 .mp4/.avi/.mov，audio/mpeg 不被支持，故直接上传视频转写，跳过 ffmpeg 抽音频。

```bash
# 1. 启动 Douyin API（8080/8081）
cd Douyin_TikTok_Download_API && python start.py

# 2. 小批量测试
python integration/run.py audio-transcribe --query-ids Q0001 --audio-batch-size 1 --audio-concurrency 1
```

**成本估算**（火山豆包 doubao-seed-2-0-pro）：

- 约 0.8 元/小时 ≈ 0.013 元/分钟
- 700 条 × 45 秒/条 ≈ 8.75 元

### 方案 C：批量 STT Pipeline（待实现）

在 `integration/pipeline.py` 或独立脚本中：

1. 筛选 `content_format=视频-无字幕` 且 `content_json.语音转文本` 为空的链接
2. 对每条：下载 → 抽音频 → 火山 ASR → 写入 `content_json.语音转文本`
3. 可加并发与限流，控制 ASR 调用频率

---

## 四、建议执行顺序

1. **验证 8080 字幕**：用本地 8080 + Cookie 调 hybrid，看 aweme_detail 是否含字幕字段
2. **若有字幕**：在 merger/crawler 中解析并落库，优先用字幕
3. **若无字幕**：对高价值/样本链接先跑 STT 测试，再决定是否批量 STT

---

## 五、成本对比小结

| 方式 | 单条成本 | 700 条估算 | 实现状态 |
|------|----------|------------|----------|
| 8080 返回 video_text（有字幕视频） | 0 | 0 | 需在 crawler 中解析 |
| 标题+简介（caption） | 0 | 0 | 已有 |
| STT 语音转文字 | ~0.01 元 | ~7 元 | audio-transcribe 已实现 |

**结论**：优先用 caption（标题/简介），有 `video_text` 时用字幕；仅当两者都无语音内容时再走 STT。
