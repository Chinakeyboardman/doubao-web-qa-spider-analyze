# 跨平台兼容性分析

## 结论概览

| 平台 | 兼容性 | 说明 |
|------|--------|------|
| **macOS** | ✅ 良好 | 文档以 Mac 为例，brew 安装 PG/Redis 最省事 |
| **Linux** | ✅ 良好 | 生产部署首选，Docker 全支持 |
| **Windows** | ⚠️ 部分兼容 | 需 WSL2 或 Docker Desktop，原生跑有坑 |

---

## 一、各子项目分析

### 1. 抖音视频下载（Douyin_TikTok_Download_API）

| 维度 | 兼容性 | 说明 |
|------|--------|------|
| 运行方式 | ✅ 跨平台 | Python + FastAPI，`os.path` 处理路径 |
| 代码 | ✅ | utils 中已区分 win32/darwin/linux 文件名长度 |
| Docker | ✅ | 官方镜像，Mac/Linux/Win 均可跑 |
| 安装脚本 | ⚠️ | `install.sh`、`update.sh` 为 bash，仅 Linux/macOS |

**建议**：优先用 Docker 部署，三端一致；若裸跑，Windows 需 WSL 或手动装 Python 后运行。

---

### 2. 豆包 web 爬虫（douyin-crawler）

| 维度 | 兼容性 | 说明 |
|------|--------|------|
| Node.js 爬虫 | ✅ | patchright 自动启动 Chrome，无硬编码路径 |
| Python Worker | ✅ | Celery + uv，跨平台 |
| 文档/脚本 | ⚠️ | README 用 `brew`、`/Applications/Google Chrome`（旧版） |
| cron 脚本 | ❌ | `scripts/run-scraper-cron.sh` 含 Mac 绝对路径、`/opt/homebrew/bin/node` |
| crontab | ⚠️ | Unix 才有，Windows 需 Task Scheduler 或第三方 |

**建议**：核心逻辑可跨平台；定时任务在 Windows 需改用计划任务或 Python 调度。

---

### 3. 待开发项目（query-input、web-crawler、data-clean）

| 维度 | 建议 |
|------|------|
| 语言 | Python 为主，跨平台 |
| 路径 | 统一用 `pathlib` 或 `os.path.join` |
| 脚本 | 避免 bash，用 Python 或跨平台脚本 |
| 换行符 | 统一 LF，避免 CRLF 导致脚本异常 |

---

## 二、依赖服务

| 服务 | Mac | Linux | Windows |
|------|-----|-------|---------|
| PostgreSQL | brew / Postgres.app | apt/yum / Docker | Docker / 官方安装包 |
| Redis | brew | apt/yum / Docker | Docker / WSL |
| Chrome | 有 | 有 | 有 |
| Docker | Docker Desktop | 原生 | Docker Desktop (WSL2) |

---

## 三、已知平台相关点

1. **brew**：仅 macOS/Linux（Homebrew on Linux）。Windows 需用 Chocolatey、Scoop 或 Docker。
2. **crontab**：仅 Unix。Windows 用「任务计划程序」或 `schedule` 等 Python 库。
3. **douyin-crawler 的 cron 脚本**：含 Mac 绝对路径，Linux/Win 需改或不用。
4. **路径分隔符**：Python/Node 用 `os.path`、`path` 模块即可，一般无问题。

---

## 四、推荐实践

| 场景 | 推荐方案 |
|------|----------|
| 开发（Mac/Linux） | 本地装 PG/Redis，直接跑各子项目 |
| 开发（Windows） | Docker 跑 PG/Redis + 抖音下载；crawler 在 WSL2 或 Docker 中跑 |
| 生产部署 | 全 Docker 编排，Linux 服务器 |
| 定时任务 | 用 Python `schedule`/Celery Beat，或系统 cron（Unix）/ 任务计划程序（Win） |

---

## 五、父项目可做的统一约定

- [ ] 新脚本用 Python 或跨平台 sh，避免依赖 bash 特性
- [ ] 路径统一用 `pathlib.Path` 或 `os.path.join`
- [ ] README 中补充 Linux/Windows 的 PG、Redis 安装说明
- [ ] `integration/` 及新业务代码避免平台相关逻辑
